SiwtuTech is fundamentally different from most web applications. Our builder is 100% client-side, meaning:

                
    
                    
  • All website generation happens entirely in your browser
    • 
                    
  • We do NOT have servers that process or store your builder data
    • 
                    
  • No accounts or logins are required to use the builder
    • 
                    
  • Your content, images, business information, and configurations never leave your device
    • 
                    
  • We cannot see, access, or recover any website you create
    • 
                

                
This means the core website builder is one of the most privacy-respecting tools available — by design, we simply cannot collect your data even if we wanted to.

Frequently Asked Questions

                
Find answers to common questions about SiwtuTech.

                
What is SiwtuTech? ▼
We are a forward-thinking digital services platform dedicated to empowering individuals and businesses through our free website builder, professional web development, cyber security services, graphics design, and custom software development.

                
How do I switch between Test and Live mode for payment gateways? ▼
When you select a payment gateway (Paystack, Flutterwave, or Stripe) and choose "API Key" as the integration type, you'll see a Test Mode / Live Mode toggle at the top. By default, it starts in Test Mode. Each mode has its own separate input field:

• Test Mode: Enter your test public key (e.g., pk_test_xxx for Paystack). A yellow warning banner reminds you that no real charges will be made.
• Live Mode: Enter your live public key (e.g., pk_live_xxx for Paystack). A green banner confirms real payments will be processed.

The key format is validated in real-time — if you accidentally paste a test key in live mode or vice versa, you'll see a warning. Only the active mode's key is used in the generated website. Start with Test Mode during development, then switch to Live Mode when deploying.

                
What key formats are validated? ▼
Paystack: Test keys must start with pk_test_, live keys with pk_live_.
Stripe: Test keys must start with pk_test_, live keys with pk_live_.
Flutterwave: Test keys must start with FLWPUBK_TEST-, live keys with FLWPUBK- (without TEST).

If the key format doesn't match the selected environment, you'll see a red or yellow validation message. If the format is correct, you'll see a green checkmark.

                
Is it safe to put my payment API key in the generated code? ▼
The keys embedded are PUBLIC keys only (e.g., pk_live_ for Paystack). Public keys are designed to be used in front-end code and are safe. Never embed secret/private keys.

                
Can I use Payment Links instead of API keys? ▼
Yes! Each gateway has an "API Key" and "Payment Link" option. With Payment Links, no API key is needed at all — customers are simply redirected to the payment provider's hosted page.

                
How does the currency system work? ▼
The admin selects ONE active display currency using a dropdown in the admin panel. Visitors only see that currency.

                
How do I access the admin panel? ▼
Click your website name/logo 5 times quickly.

                
Is the builder really free? ▼
Yes! The DIY website builder is 100% free.
